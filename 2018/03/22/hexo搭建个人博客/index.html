利用github + hexo搭建个人博客

2018-03-22

前两天一同学把他的博客地址发在我们的班级群里面，让我们去观摩他的技术文档，说是买个服务器自己搭在服务器上，然后群里突然就掀起了一阵服务器热，大家都去买（正好腾讯云打折，一年也就120RMB）。虽然我是买了服务器，不过我的都是用来搭建SS和v2ray的（配置贼烂）。后来同事让我去用GitHub搭建一个，想想也是写给自己和同学同事看的也就没必要搞那么复杂了，就去通过GitHub Page + hexo搭建了一个，这篇文章介绍一下我的搭建过程（虽然百度有很多，不过还是想用自己的语言来组织），整个过程还算顺利，第一次写文章，如果有写的不好的地方 请多指正 谢谢。

搭建步骤

  • 安装nodejs nodejs官网地址
  • 安装git git下载地址
  • GitHub新建仓库（New Repositories）
  • 配置GitHub SSH密钥
  • 安装hexo

安装nodejs/git

为什么安装nodejs（因为hexo基于nodejs开发）直接去官网下载即可所以前两个步骤就不多赘述了,都是一直下一步（百度很多，有些配置可以自己改，我都是下一步）.安装完成之后cmd输入命令 git/node –version,如图1所示即正确安装，如果报图2的错，应该就是环境变量没有配置，将安装的目录地址配置到path里面再试一次。


新建GitHub仓库（New Repositories）

注册一个GitHub，有的就直接开始吧。

登陆之后 点击New Repositories

repository name填写自己的GitHub的用户名.github.io（也就是你以后的博客名了），Description随便写一个吧，readme可要可不要。点击Create repository。

配置GitHub密钥

桌面空白处右键Git Bash Here

ssh-keygen -t rsa -C "your's emaill address"

会提示你保存的路径回车即可，输入密码密码看不见，完成之后会生成两个文件，id_rsa和id_rsa.pub，用编辑器打开id_rsa.pub选中里面的全部内容复制下来。

登录GitHub点击头像下拉列表有个settings点击进入，左侧树有个SSH and GPG keys选中

按照图示完成操作 SSH配置即完成。记得设置用户名和邮箱

$ git config --global user.name "Your's name"
$ git config --global user.email "Your's email address"

Your’s name是你的GitHub用户名，Your’s email address是你的邮箱地址

安装hexo

nodejs和Git完成之后我们可以来安装hexo了，桌面任意位置右键鼠标选择 Git Bash Here或者进入Git安装目录双击git-bash.exe。

安装hexo，等待安装完成

npm install -g hexo-cli 。

新建一个空文件夹用于个人博客

hexo inio

配置_config.yml配置文件（为了快速搭建成功，只修改deploy部分，其余部分以后再出一篇修改配置文件的，目前知道的不多），github用户名当然是改成自己的了。文件在你刚刚创建的文件夹下，懂Linux的也可以直接vim改，改完保存。

deploy:
      type: git
      repository: https://github.com/wisemanjie/wisemanjie.github.io.git
      branch: master

在你创建的文件夹下打开git-bash输入命令依次输入以下命令

$ hexo clean
$ hexo g
$ hexo s

现在本地看看效果 https://localhost:4000，如果配置没有出错，你会看见效果的（应该还是会很开心的）。接下来就是部署到github上面依次输入以下命令

$ hexo clean
$ hexo g
$ hexo d

博客地址为

http://yourname.github.io

到这里所有的配置及部署已完成，本篇幅只介绍步骤，如有问题可以留言或者等我出一篇遇到的问题或者去百度、Google，相信你很快就会看到自己辛苦的成果（给自己一个赞吧）。
