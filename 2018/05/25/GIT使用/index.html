GIT使用

2018-05-25

  • 新建版本库

新建文件夹（最好为空，文件名不要有中文）
文件夹空白处鼠标右键Git Bash Here
命令行输入git init如不发生错误文件夹内会多出一个.git文件夹（看不到文件夹选项打开隐藏文件权限即可）。新建文件放入本地库（最好别用Windows自带的文本编辑器）

git add    本地库新增文件    

git commit -m <message>    本地库文件提交

git status    随时掌握工作区状态

git diff    查看修改内容

git log    (--pretty=oneline)    查看提交历史(类似一串1094ad...的是commit id)

git reset (--hard commit id)（版本号可以不写全） 上个版本HEAD^上上个版本HEAD^^网上100个版本HEAD~100

git reflog        查看命令历史
  • 工作区和暂存区

工作区指你本地自己创建的目录，工作区目录下有个.git（隐藏目录），它是Git的版本库，里面存了很多东西 最重要的是成为stage或者index的暂存区。还有Git为我们创建的第一个分支master，志向master的指针HEAD

文件往Git版本库里添加的时候分为两步执行：

第一步git add把文件添加进去，实际上就是把文件修改添加到暂存区；

第二步git commit提交更改，把暂存区的所有内容提交到当前分支；

  • 撤销修改
    git checkout – file（可以丢弃工作区的修改，–很重要，没有就是另一种切换到另一分支的命令）

分为两种情况：

第一种文件自修改后还没有放到暂存区，现在撤销修改会回到和版本库一模一样的状态。

第二种文件已经添加到暂存区，又做了修改，现在撤销修改就回到添加到暂存区的状态。

  • 删除文件

如果要从版本库删除文件 git rm file git commit

  • git远程库

先要进行sshkey绑定（自行百度）

关联远程库 git remote add origin git@github.com:michaelliao/learngit.git

第一次推送 git push -u origin master

以后每次推送 git push origin master

由于远程库是空的第一次加上了-u参数，推送并关联远程库，以后可以简化命令

  • 分支管理

    查看分支 git branch

    创建分支 git branch

    切换分支 git checkout

    创建和切换分支 git checkout -b

    合并某分支到当前分支 git merge

    删除分支 git branch -d

解决冲突

git log --graph    查看分支合并图

合并分支时加上--no-ff参数就可以使用普通模式合并，合并后的历史有分支，可以看出来曾经做过合并

Bug分支

修复bug时，创建新的分支修改，然后合并 删除分支；

手头工作没有完成的时候可以git stash 存储；修复后，在git statsh pop 删除；git stash list 查看存储的工作区； git stash apply stash@{0}恢复指定的stash。

多人协作

工作模式：

1、git push origin <branch-name>推送自己的修改

2、推送失败，应该是远程分支比你的本地更新，需要git push试图合并

3、合并有冲突，则解决冲突，并在本地提交

4、没有冲突或者冲突解决之后，git push origin <branch-name>

如果git pull提示no tracking infomation,则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream-to <branch-name> origin/<branch-name>。

git remote -v    查看远程库信息
git checkout -b branch-name origin/branch-name 在本地创建和远程分支对应的分支，名称最好一样
git branch --set-upstream branch-name origin/branch-name    简历本地分支和远程分支的关联

远程抓取分支git pull,有冲突先解决冲突

-标签tag

git tag <tagname>用于新建一个标签，默认为HEAD，也可以制定一个commit id;

git tag -a <tagname> -m ""可以指定标签信息;

git tag查看所有标签。

git show <tagname>查看标签信息

如果没有推送到远程库，Delete tag <tagname>删除标签，如果已经推送到远程库，先从本地删除，然后从远程删除 git push origin:refs/tags/<tagname>

git push origin <tagname>推送本地标签

git push origin --tags可以推送全部未推送过的本地标签

在github上可以Fork任意开源仓库，Fork之后获得仓库的读写权限，

git config --global color.ui true 使Git有颜色

  • 搭建Git服务器

准备Linux服务器（Ubuntu和Debian最佳）

如果已经有sudo权限 sudo apt-get install git 安装git sudo adduser git 创建一个git用户

把所有用户的公钥导入 /home/git/.ssh/authorized_keys文件里，一行一个

初始化Git仓库 sudo git init --bare <name>.git

将owner改为Git sudo chown -R git:git name.git

禁用shell登录git:x:1001:1001:,,,:/home/git:/bin/bash替换成git:x:1001:1001:,,,:/home/git:/usr/bin/git-shell

git clone git@server:/srv/name.git

要方便管理密钥 用Gitosis,更严格的Gitolite
